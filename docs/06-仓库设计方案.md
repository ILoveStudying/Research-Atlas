# research-atlas 仓库设计方案

## 设计目标

`research-atlas` 服务长期学术调研。它既要能容纳跨领域知识地图，也要能落到具体 topic 的论文、代码、配置和复现实验。

## 架构

顶层仓库负责路由、治理、索引和自动记录。每个 topic 负责具体研究闭环，并拥有独立的 `llm-wiki`。

```text
顶层 registry
  -> domain
    -> track
      -> subfield
        -> topic
          -> papers / code / synthesis / llm-wiki
```

## 自动更新记录

自动记录由脚本完成。任何新增入口脚本最后都调用：

```bash
bash scripts/record-update.sh <类型> <范围> <标题> <路径> <说明>
```

这会同步追加 Markdown 日志和 YAML 日志。

## 初始 topic

初始 topic 是 `st-data-tokenization`，用于承载“大模型时空世界 Token 化”方向的论文、代码和综合材料。
