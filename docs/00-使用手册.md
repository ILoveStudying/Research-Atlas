# 使用手册

## 一句话原则

所有研究内容最终都落到 topic。每个 topic 独立维护论文、代码、配置、实验记录、综合文档和 `llm-wiki`。

## 新增 topic

使用脚本创建 topic，脚本会生成基础目录、元数据文件，并初始化独立 `llm-wiki`：

```bash
bash scripts/create-topic-wiki.sh <domain> <track> <subfield> <topic> "<topic中文名>"
```

示例：

```bash
bash scripts/create-topic-wiki.sh ai spatiotemporal-foundation-models representation-and-tokenization st-data-tokenization "时空数据 Token 化"
```

## 添加论文

论文记录放在 topic 内的 `papers/records/`，精读笔记放在 `papers/notes/`，PDF 可放在 `papers/pdfs/` 或只登记外部链接。

推荐入口：

```bash
bash scripts/add-paper-record.sh <topic路径> <论文id> "<标题>" <年份> <会议> <分级> "<论文链接>" "<代码链接>"
```

## 添加代码

代码记录放在 topic 内的 `code/repos/`，关键配置快照放在 `code/configs/`，复现记录放在 `code/runs/`。

推荐入口：

```bash
bash scripts/add-code-record.sh <topic路径> <代码id> "<项目名>" <论文记录路径> "<仓库链接>" <主要框架>
```

## 消化素材到 llm-wiki

进入 topic 的 `llm-wiki/` 后，将论文、网页、PDF、代码说明或实验记录交给 agent 消化。该 wiki 的 `purpose.md` 会约束知识组织方向。
