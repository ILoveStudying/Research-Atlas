# 研究分类体系

## 四级路由

```text
domain -> track -> subfield -> topic
```

- `domain`：研究大领域，例如 AI、系统、机器人、科学智能。
- `track`：领域内稳定方向，例如时空大模型、世界模型、多模态大模型。
- `subfield`：方向下的问题簇，例如表示学习与 Token 化、评测与 Benchmark。
- `topic`：最终工作单元，必须能承载论文、代码、配置、实验和 wiki。

## 为什么 topic 是最终落点

论文和代码通常不是只属于一个宽泛方向，而是服务某个具体研究问题。把资产落到 topic，可以让后续检索、复现、对比和写综述更直接。

```text
AI

```

## 新增分类规则

新增分类时优先判断它是否能独立积累 10 篇以上论文或 3 个以上代码仓库。如果不能，先作为已有 topic 下的实体或主题页维护。
