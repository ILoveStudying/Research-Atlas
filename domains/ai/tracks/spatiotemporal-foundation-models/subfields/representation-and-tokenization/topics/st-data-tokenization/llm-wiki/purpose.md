# 研究目的与方向

## 核心目标
系统追踪真实世界时空数据如何被 token 化、序列化、离散化或 prompt 化，并比较这些表示如何服务大模型、世界模型、预测模型和工具调用。

## 关键问题
1. 不同论文如何定义时空 token、ST-unit、grid token、road segment token、sensor token、weather prompt、event token 和 action token？
2. 这些 token 中包含哪些空间、时间、拓扑、动态属性、语义事件、物理约束和置信度信息？
3. tokenization 与下游任务之间是什么关系，例如预测、生成、检索、规划、问答和仿真？
4. 哪些方法可以跨城市、跨模态、跨任务、跨时间分布泛化？
5. 论文代码中哪些配置、数据预处理和模型接口决定了时空表示质量？

## 研究范围
**涵盖：** 城市交通、人流轨迹、自动驾驶、遥感、天气、视频世界模型中与时空表示和 token 化直接相关的论文、代码、配置和复现实验。
**不涵盖：** 只做传统单任务预测、没有基础模型或统一表示接口的工作；仅作为背景材料时可以登记，但不作为核心精读对象。
